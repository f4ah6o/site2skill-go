This page exists to verify the --include and --exclude options. Use it to ensure filters are applied consistently before converting documentation into skills.

How to Test

# Crawl only filter pages and skip the excluded one
site2skillgo generate --include "filters" --exclude "exclude" https://f4ah6o.github.io/site2skill-go/ filter-playground

# Verify exclude wins over include
site2skillgo generate --include "filters,exclude" --exclude "exclude" https://f4ah6o.github.io/site2skill-go/ filter-playground

Test URLs

Path Expectation
/docs/filters/include-only/ Fetched only when include filter matches "filters" or "include"
/docs/filters/exclude-me/ Must be skipped when exclude filter includes "exclude"
/docs/filters/mixed-signal/ Always fetched (control page)

Why This Matters

  • Include filters keep crawls focused on specific versions or sections.
  • Exclude filters prevent unwanted content (beta pages, admin paths) from being packaged.
  • Exclude overrides include to avoid accidental leakage when patterns overlap.
